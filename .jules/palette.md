## 2024-05-02 - Actionable Empty States
**Learning:** Empty states for dynamic user-generated content (like history or session lists) that simply state "No data" create a dead-end for users, reducing engagement. In this app, the lack of data indicates a new user who needs guidance on what to do next.
**Action:** Always implement empty states as actionable, visually distinct cards (e.g., using dashed borders) with a clear icon, explanation, and a direct call-to-action button linking to the relevant part of the application (e.g., Skills Hub) to encourage immediate engagement.
