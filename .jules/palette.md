## 2024-05-24 - Actionable Empty States
**Learning:** Simple text-based empty states (e.g., "No history yet", "No skills found") fail to encourage user engagement and can make the application feel like a dead end.
**Action:** Always implement empty states as visually distinct cards that include a helpful icon, descriptive text, and, critically, a clear call-to-action button (like "Explore Skills" or "Clear Filters") that gives the user an immediate path forward.
