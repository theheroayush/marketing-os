## 2026-04-28 - Actionable Empty States
**Learning:** Users encountering static text "empty states" (e.g., "No history yet", "No skills found") often feel stuck without a clear next step. Providing a direct call-to-action button within the empty state itself bridges the gap between intention and action, significantly improving engagement and reducing friction.
**Action:** Always include an actionable button (like "Explore Skills" or "Clear Search") when rendering empty dynamic content blocks to guide the user to the logical next step.
